# Custom Renderer Plugins

This document describes how to create custom renderer plugins for the Art Engine.

## Interface

All renderer plugins in the Art Engine implement the `RendererInterface<RendererDataType>` interface:

```typescript
interface RendererInterface<RendererDataType> {
  init: (props: RendererInitPropsInterface) => Promise<void>;
  render: () => Promise<ItemsRenders<RendererDataType>>;
}
```

- `RendererInterface<RendererDataType>`: This interface outlines the structure that renderer plugins should follow. It's parameterized with a generic type `RendererDataType` which represents the type of data that the renderer plugin will produce.
- `init(props: RendererInitPropsInterface) => Promise<void>`: This method initializes the renderer plugin. It takes an object of type `RendererInitPropsInterface` as an argument, containing the necessary initialization properties. The method returns a promise that resolves when the initialization is complete.
- `render() => Promise<ItemsRenders<RendererDataType>>`: This method is responsible for performing the rendering process. It returns a promise that resolves to an object of type `ItemsRenders<RendererDataType>`, which is the rendered data produced by the plugin.

### ItemsRenders

```typescript
interface ItemsRenders<RenderDataType> {
  [itemUid: string]: ItemPropertiesInterface<RenderDataType>[];
}
```

This is a data structure that represents the renders produced by a renderer plugin. It's an object where the keys are item UIDs (unique identifiers) and the values are arrays of render data of type `RenderDataType`. This structure allows you to store multiple render data sets associated with different items.

### RendererInitPropsInterface

```typescript
interface RendererInitPropsInterface {
  seed: string;
  cachePath: string;
  attributesGetter: ItemsDataManager["getAttributes"];
}
```

- `seed`: A string value that serves as a seed for deterministic behavior. Similar to the input plugins, this can be used to ensure consistent results when rendering.
- `cachePath`: A string representing the path where the renderer plugin can store temporary cache data. This can be used to store any intermediate files or data that the renderer might need to use.
- `attributesGetter`: A reference to a function called `getAttributes` from the `ItemsDataManager`. This function is used to access the attributes data generated by the generators.

### ItemPropertiesInterface

Each render in the array must have a `kind` and `data` property:

```typescript
{
  kind: "YourUniqueRenderIdentifier@v1",
  data: {
    // Your custom render data structure
  }
}
```

- `kind`: A unique string identifier for this type of render (e.g., `"MyCustomRender@v1"`)
- `data`: The actual render data matching your `RendererDataType`

## Example Implementation

```typescript
import * as path from "path";
import * as fs from "fs";
import RendererInterface, {
  ItemsRenders,
  RendererInitPropsInterface,
} from "@hashlips-lab/art-engine/dist/common/renderers/renderer.interface";
import ItemsDataManager from "@hashlips-lab/art-engine/dist/utils/managers/items-data/items-data.manager";
import { CACHE } from "@hashlips-lab/art-engine";
import { ItemPropertiesInterface } from "@hashlips-lab/art-engine/dist/utils/managers/items-data/items-data.interface";

// Define the structure of the custom render data
interface ExampleCustomInterface {
  path: string;
}

// Create a class for the Renderer plugin
export class ExampleRenderer
  implements RendererInterface<ExampleCustomInterface>
{
  attributesGetter!: ItemsDataManager["getAttributes"];
  tempRenderDir!: string;

  public async init(props: RendererInitPropsInterface): Promise<void> {
    this.attributesGetter = props.attributesGetter;
    this.tempRenderDir = path.join(
      props.cachePath,
      CACHE.RENDERERS_TEMP_CACHE_DIR,
      "words"
    );
    // Initialization tasks can be performed here
  }

  public async render(): Promise<ItemsRenders<ExampleCustomInterface>> {
    const renders: ItemsRenders<ExampleCustomInterface> = {};

    for (const [itemUid, attributes] of Object.entries(
      this.attributesGetter()
    )) {
      if (!fs.existsSync(this.tempRenderDir)) {
        fs.mkdirSync(this.tempRenderDir, { recursive: true });
      }
      
      // Filter attributes by kind to find the ones this renderer processes
      const supportedAssets = attributes.filter(
        (attribute: ItemPropertiesInterface<ExampleCustomInterface>) =>
          "AnyUniqueGeneratorDataIdentifier@v1" === attribute.kind
      );

      if (supportedAssets.length > 0) {
        const outputPath = path.join(this.tempRenderDir, `${itemUid}.json`);
        fs.writeFileSync(
          outputPath,
          JSON.stringify(supportedAssets[0].data, null, 2)
        );

        renders[itemUid] = [
          {
            kind: "AnyUniqueRenderDataIdentifier@v1",
            data: {
              path: outputPath,
            },
          },
        ];
      }
    }

    return renders;
  }
}
```

## Implementation Steps

1. **Import Required Interfaces**:
   ```typescript
   import RendererInterface, {
     ItemsRenders,
     RendererInitPropsInterface,
   } from "@hashlips-lab/art-engine/dist/common/renderers/renderer.interface";
   import ItemsDataManager from "@hashlips-lab/art-engine/dist/utils/managers/items-data/items-data.manager";
   import { CACHE } from "@hashlips-lab/art-engine";
   import { ItemPropertiesInterface } from "@hashlips-lab/art-engine/dist/utils/managers/items-data/items-data.interface";
   ```

2. **Define Your Render Data Type**:
   Create an interface that represents the structure of data your renderer will produce.

3. **Implement the Interface**:
   - Create a class that implements `RendererInterface<YourDataType>`
   - Store `attributesGetter` and `cachePath` from `init()`
   - Set up temporary directory using `CACHE.RENDERERS_TEMP_CACHE_DIR`
   - Implement the `init()` method for any setup tasks
   - Implement the `render()` method to process attributes and create renders
   - Filter attributes by `kind` to find the ones your renderer processes
   - Return an object mapping item UIDs to arrays of renders with `kind` and `data`

4. **Use Unique Kind Identifiers**:
   Each render must have a unique `kind` identifier (e.g., `"MyCustomRender@v1"`)

5. **Place in Custom Directory**:
   Save your plugin in `custom/renderers/your-renderer.ts`

6. **Use in Configuration**:
   ```typescript
   const { ExampleRenderer } = require("./custom/renderers/example-renderer");
   
   const ae = new ArtEngine({
     renderers: [
       new ExampleRenderer(),
     ],
     // ... rest of config
   });
   ```

## Accessing Attributes

Use the `attributesGetter` function to access attributes generated by generators:

```typescript
// Get all attributes for all items
const allAttributes = this.attributesGetter();

// Iterate over items
for (const [itemUid, attributes] of Object.entries(allAttributes)) {
  // Process attributes for this item
  // Filter by kind to find relevant attributes
  const relevantAttributes = attributes.filter(
    (attr) => attr.kind === "YourGeneratorKind@v1"
  );
}
```

## Temporary Storage

Use the `cachePath` and `CACHE.RENDERERS_TEMP_CACHE_DIR` to store temporary files:

```typescript
const tempDir = path.join(
  props.cachePath,
  CACHE.RENDERERS_TEMP_CACHE_DIR,
  "your-plugin-name"
);
```

## Use Cases

Custom renderer plugins can be used to:
- Render images from attributes
- Generate metadata files
- Create animations
- Process audio/video
- Generate 3D models
- Any other rendering logic

## Interface Capabilities

The `RendererInterface<RendererDataType>` interface is **completely flexible**. The `RendererDataType` generic parameter can be **any TypeScript type or interface**, allowing you to:

- **Render Any Media Type**: Images, videos, audio, 3D models, animations, documents, or any other media
- **Use Any Rendering Logic**: The `render()` method can use image processing, video editing, audio synthesis, 3D rendering, AI generation, or any other rendering technique
- **Access Any Attributes**: Use `attributesGetter()` to access attributes from any generator plugin, then filter by `kind` to find the ones you need
- **Store Any Render Data**: The render data structure can contain file paths, URLs, metadata, or any information exporters need

### Examples of What's Possible

The interface supports rendering:
- **Images**: Layered images, AI-generated images, processed images, animated GIFs, WebP animations
- **Videos**: Video sequences, edited videos, composited videos, animated videos, video effects
- **Audio**: Music generation, sound synthesis, audio mixing, audio effects
- **3D Models**: 3D model generation, scene rendering, model variations, 3D animations
- **AI Content**: Text-to-image, image-to-image, video generation, audio generation
- **Metadata**: JSON files, XML files, custom metadata formats
- **Any Other Media**: Documents, animations, interactive content, or any other format

### Using External Libraries

You can use **any npm package** in your renderer plugin. For example:
- **Video**: `fluent-ffmpeg`, `ffmpeg-static` for video processing
- **Image**: `sharp`, `canvas`, `jimp` for image manipulation
- **AI**: `openai`, `@stability-ai/sdk`, `@replicate/replicate-js` for AI rendering
- **3D**: `three`, `@react-three/fiber` for 3D rendering
- **Audio**: `wavefile`, `tone` for audio processing
- **APIs**: `axios`, `node-fetch` for external service calls
- Any other package you need

Simply install the package and import it in your plugin file.

### Temporary File Storage

The `cachePath` and `CACHE.RENDERERS_TEMP_CACHE_DIR` are provided for storing temporary files:

```typescript
const tempDir = path.join(
  props.cachePath,
  CACHE.RENDERERS_TEMP_CACHE_DIR,
  "your-plugin-name" // Use a unique name for your plugin
);
```

Store any intermediate files here (images, videos, audio, 3D models, etc.) and include the paths in your render data for exporters to use.

### Processing Multiple Attributes

You can process attributes from multiple generators:

```typescript
for (const [itemUid, attributes] of Object.entries(this.attributesGetter())) {
  const imageAttrs = attributes.filter(attr => attr.kind === "ImageAttributes@v1");
  const videoAttrs = attributes.filter(attr => attr.kind === "VideoAttributes@v1");
  // Process and combine attributes as needed
}
```

## Important Notes

- The `kind` identifier must match the `kind` from the generator attributes you want to process
- Each item UID should map to an array of renders
- Each render must have `kind` (string) and `data` (your `RendererDataType`) properties
- Renderers run after generators and before exporters
- Multiple renderers can be used, and their outputs are combined
- Use `cachePath` for temporary files that exporters will use
- The `kind` identifier is used by exporters to identify which renders to process
- Always create directories with `recursive: true` if they don't exist
- The `RendererDataType` can be **any TypeScript type** - define it to match your needs
- You can use any npm package - there are no restrictions on dependencies
- The `render()` method is async, so you can perform any asynchronous operations (file I/O, API calls, processing, etc.)
